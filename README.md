# LectoTech

## 📌 Objetivo de la Aplicación
LectoTech es una plataforma digital educativa integral diseñada para fortalecer las habilidades de lectoescritura en estudiantes de Educación General Básica (5to a 7mo grado). La aplicación utiliza un enfoque neuroeducativo con psicología del color para reducir la fatiga visual, aumentar la motivación y fomentar el aprendizaje interactivo a través de módulos gamificados (juegos, lectura y talleres de escritura).

## 🏗️ Arquitectura
La aplicación sigue una arquitectura moderna Cliente-Servidor (Client-Server) basada en componentes y orientada al Server-Side Rendering (SSR) y Static Site Generation (SSG) gracias a Next.js.
- **Frontend (Cliente):** Interfaz de usuario responsiva e interactiva construida con React y Tailwind CSS, con ruteo de Next.js (App Router).
- **Backend as a Service (BaaS):** Supabase maneja la autenticación, base de datos (PostgreSQL), almacenamiento de archivos y políticas de seguridad (RLS).
- **Despliegue Continua (Hostinger / GitHub):** El repositorio de GitHub está configurado para permitir el despliegue automático hacia plataformas como Hostinger, de modo que cada cambio local subido a GitHub se refleje automáticamente en línea.

## 🛠️ Tecnologías Utilizadas
- **Framework Principal:** [Next.js](https://nextjs.org/) (React)
- **Estilos:** [Tailwind CSS](https://tailwindcss.com/)
- **Base de Datos & Backend:** [Supabase](https://supabase.com/) (PostgreSQL)
- **Lenguaje:** TypeScript / JavaScript
- **Control de Versiones:** Git & GitHub

## 📊 Esquema de Base de Datos (Supabase)
La plataforma utiliza un esquema relacional optimizado en PostgreSQL a través de Supabase:

| Tabla | Funcionalidad | Descripción de Datos |
| :--- | :--- | :--- |
| `profiles` | Perfiles de Usuario | Almacena el nombre real, correo, rol (docente/estudiante), código de clase y avatar. |
| `messages` | Centro de Comunicación | Gestiona los mensajes (públicos de clase o privados), URLs de adjuntos (PDF/Imágenes), YouTube links y estado de archivado. |
| `message_stars` | Gamificación (Likes) | Registra qué estudiantes han marcado con una estrella los mensajes del profesor. |
| `message_comments` | Feedback Interactivo | Permite a los estudiantes comentar en los mensajes para resolver dudas. |
| `library_books` | Biblioteca Digital | Contiene los libros PDF asignados por el profesor a códigos de clase específicos. |
| `activities_scores` | Seguimiento Académico | Registra los puntos y el progreso de los estudiantes en los diferentes juegos y retos. |

## 🛠️ Configuración de Base de Datos (SQL)
Para replicar este proyecto en una nueva cuenta de Supabase, ejecuta los siguientes scripts en el **SQL Editor**:

### 1. Creación de Tablas Principales
```sql
-- Tabla de Perfiles (Sincronizada con Auth)
CREATE TABLE public.profiles (
  id UUID REFERENCES auth.users ON DELETE CASCADE PRIMARY KEY,
  nombre TEXT,
  email TEXT,
  rol TEXT DEFAULT 'estudiante',
  grado TEXT,
  avatar_url TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Tabla de Mensajes Multimedia
CREATE TABLE public.messages (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  sender_id UUID REFERENCES auth.users(id),
  target_id TEXT, -- Puede ser un grado (5to, 6to) o el ID de un estudiante
  target_type TEXT, -- 'class' o 'private'
  content TEXT,
  attachment_url TEXT,
  youtube_url TEXT,
  is_archived BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Tabla de Biblioteca Digital
CREATE TABLE public.library_books (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  title TEXT,
  pdf_url TEXT,
  cover_url TEXT,
  assigned_to TEXT, -- Grado asignado
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Gamificación y Feedback
CREATE TABLE public.message_stars (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  message_id BIGINT REFERENCES public.messages(id) ON DELETE CASCADE,
  user_id UUID REFERENCES auth.users(id),
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE public.message_comments (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  message_id BIGINT REFERENCES public.messages(id) ON DELETE CASCADE,
  user_id UUID REFERENCES auth.users(id),
  content TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

### 2. Sincronización Automática de Usuarios (Trigger)
```sql
-- Función para crear el perfil automáticamente al registrarse
CREATE OR REPLACE FUNCTION public.handle_new_user()
RETURNS TRIGGER AS $$
BEGIN
  INSERT INTO public.profiles (id, email, nombre, grado, rol)
  VALUES (
    NEW.id,
    NEW.email,
    COALESCE(NEW.raw_user_meta_data->>'nombre', 'Estudiante Nuevo'),
    COALESCE(NEW.raw_user_meta_data->>'grado', 'Sin Grado'),
    COALESCE(NEW.raw_user_meta_data->>'rol', 'estudiante')
  );
  RETURN NEW;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

-- Trigger
CREATE TRIGGER on_auth_user_created
  AFTER INSERT ON auth.users
  FOR EACH ROW EXECUTE FUNCTION public.handle_new_user();
```

## 👥 Roles de Usuario
El sistema diferencia dos tipos de acceso con interfaces personalizadas:
1. **👨‍🏫 Docente (Guía):** Administrador del sistema. Acceso mediante código `MAESTRIA-UG`.
2. **👦 Estudiante (Explorador):** Usuario final. Acceso mediante códigos `5TO-CLASE`, `6TO-CLASE`, `7MO-CLASE`.

## 🚀 Cómo Configurar y Ejecutar Localmente

### 1. Prerrequisitos
- [Node.js](https://nodejs.org/) (versión 18 o superior)
- Git instalado en tu sistema
- Una cuenta y proyecto en [Supabase](https://supabase.com/)

### 2. Clonar el Repositorio
```bash
git clone https://github.com/lookchatec-arch/LectoTech.git
cd LectoTech
```

### 3. Instalar Dependencias
```bash
npm install
```

### 4. Configurar Variables de Entorno
Crea un archivo `.env.local` en la raíz del proyecto (o usa el que ya viene configurado) y agrega tus credenciales de Supabase:
```env
NEXT_PUBLIC_SUPABASE_URL=tu-url-de-proyecto
NEXT_PUBLIC_SUPABASE_ANON_KEY=tu-anon-key
```

### 5. Iniciar el Servidor de Desarrollo
```bash
npm run dev
```
Abre [http://localhost:3000](http://localhost:3000) en tu navegador para ver la aplicación.
